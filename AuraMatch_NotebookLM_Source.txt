# AuraMatch — AI-Powered Beauty Recommendation Platform
# Complete Project Documentation for NotebookLM
# Last updated: 2026-04-09

## 1. Project Overview

AuraMatch is a full-stack AI-powered beauty/cosmetics recommendation platform. It analyzes facial structure and personal color to recommend makeup products.

**Core Features:**
- Face shape detection (7 shapes: oval, round, square, heart, oblong, diamond, triangle)
- Skin tone analysis (6 levels: fair/light/medium/tan/dark/deep) using ITA formula
- Undertone classification (warm/cool/neutral) using CIELAB hue angle
- Personal color season classification (4 seasons, 12 sub-seasons via CIEDE2000)
- 6-signal scoring engine for product recommendations (S1–S6)
- Product catalog with compare feature (up to 3 products, color match % via S2+S4)
- Real-time photo editor with makeup try-on
- AI chat via Google Gemini for beauty advice
- Affiliate commission tracking (Shopee, TikTok, Lazada, Sephora, Watsons, Ulta)
- User behavior analytics with event tracking
- Blog, favorites, reviews, skin concerns, banners

**Tech Stack:**
- Frontend: React 19 + Vite 8, Tailwind CSS 4, React Router DOM 7, Axios
- Backend: Python FastAPI, SQLAlchemy 2.0, MySQL (PyMySQL)
- AI/ML: OpenCV, MediaPipe FaceMesh (468 landmarks), dlib (68 landmarks), TensorFlow/Keras (CNN), Google Gemini 2.0
- Color Science: colour-science library (CIELAB, CIEDE2000), scikit-learn (K-Means)
- Auth: JWT (HS256) with bcrypt password hashing

## 5. Core AI/ML Pipeline

### Face Analysis Pipeline (Single-Pass Architecture)
analyze_face() in ai_service.py runs ONE MediaPipe FaceMesh session:
1. _run_facemesh(rgb_image) → returns landmarks (468 points), face bounding box, face crop
2. detect_face_shape(rgb_image, facemesh_data) → CNN primary path or geometry fallback
3. extract_skin_lab_from_bbox(rgb_image, bbox) → K-Means clustering (3 clusters) on cheek region
4. classify_tone_from_lab(L*, b*) → ITA formula → 6 tone classes
5. classify_undertone_from_lab(a*, b*) → hue angle → warm/cool/neutral
6. match_season_by_delta_e(skin_lab, palette_refs) → season + palette_id + confidence

Landmark coordinates are converted to tuples before exiting MediaPipe context manager (underlying data becomes invalid after context exits).

### Face Shape Detection
**CNN Path (Primary):** Model at models_ai/face_shape_cnn.h5, produces all 7 shapes including triangle.
**Geometry Fallback:** Uses MediaPipe landmark ratios:
- ratio = face_height / cheekbone_width (landmarks 152↔10 / 234↔454)
- jaw_ratio = jaw_width / cheekbone_width (landmarks 172↔397 / 234↔454)
- temple_ratio = temple_width / cheekbone (landmarks 127↔356 / 234↔454)

Decision tree (Farkas 1994 anthropometric thresholds):
- ratio > 1.45 → oblong (hyperleptoprosopic)
- 1.30 < ratio ≤ 1.45:
  - jaw_ratio < 0.70 → heart
  - temple_ratio < 0.82 → diamond
  - else → oval (leptoprosopic)
- 1.15 < ratio ≤ 1.30:
  - jaw_ratio > 0.85 → square
  - jaw_ratio < 0.70 → heart
  - temple_ratio < 0.82 → diamond
  - else → oval (mesoprosopic)
- ratio ≤ 1.15:
  - jaw_ratio > 0.85 → square
  - else → round (euryprosopic)
- no face → oval (default)

Reference: Farkas, L. G. (1994). Anthropometry of the Head and Face (2nd ed.). Raven Press.
Farkas normative data: total facial index mean 1.28 (SD ~0.06), mandibular-bizygomatic mean 0.71 (SD ~0.04), frontal-bizygomatic mean 0.87 (SD ~0.03).

### Color Science Rules
1. **Always use CIELAB (L*a*b*) color space** — never raw RGB for color analysis. RGB is device-dependent and perceptually non-uniform.
2. **CIEDE2000 (ΔE00)** for all color distance calculations — not CIE76 or CIE94. Reference: Sharma, G., Wu, W., & Dalal, E. N. (2005). Color Research & Application, 30(1), 21-30.
3. **Skin tone via ITA (Individual Typology Angle):** ITA = arctan((L* - 50) / b*) × 180/π. Thresholds: >55° fair, >41° light, >28° medium, >10° tan, >-30° dark, ≤-30° deep. Reference: Chardon, A., et al. (1991). Int J Cosmet Sci, 13(4), 191-208. Del Bino, S. & Bernerd, F. (2013). Br J Dermatol, 169(s3), 33-40.
4. **Undertone via CIELAB hue angle:** h_ab = atan2(b*, a*) in degrees. >65° warm, <50° cool, 50-65° neutral. Never use raw a*/b* thresholds. Reference: Xiao, K., et al. (2017). Skin Res Technol, 23(1), 21-29. Ly, B. C. K., et al. (2020). J Invest Dermatol, 140(1), 3-12.
5. **Season matching uses Delta E distance** against color_palettes.best_colors.skin_reference centroids. Not hardcoded lookup dicts.
6. **Palette data stored in CIELAB** with {"L", "a", "b"} format in JSON columns.
7. **colour-science library** (import colour) is the canonical dependency. RGB→CIELAB path: sRGB → XYZ (D65) → L*a*b*.

### Skin Extraction
- extract_skin_lab_from_bbox(): Uses face bounding box to sample cheek region (middle third vertically, inner half horizontally)
- Filters outlier pixels: L* < 20 (hair/eyebrows) or L* > 95 (highlights)
- K-Means Clustering (3 clusters) → largest cluster = dominant skin color
- Falls back to extract_skin_lab() (center-crop) when no face detected

## 6. Recommendation Scoring Engine (S1–S6)

Products are ranked by a weighted composite score (0–999.99) from 6 signals:

| Signal | Range | Description |
|--------|-------|-------------|
| S1 | 0–300 | Rule Match Priority — base priority from rule-based matching (face_shape, skin_tone, undertone, personal_color, gender) |
| S2 | 0–300 | CIEDE2000 Proximity — closeness of product color shades to palette best_colors |
| S3 | 0–150 | Seasonal Match — bonus for matching user's personal color season (150=match, 0=mismatch, 75=no data) |
| S4 | -100–0 | Avoid-Color Penalty — deduction when product shade close to avoid-color (delta E < 15) |
| S5 | 0–100 | Feedback Aggregation — like/dislike ratio from users with same personal color |
| S6 | 0–50 | Popularity — favorites count (capped at 10 favorites = 50 points) |

Final Score = clamp(S1 + S2 + S3 + S4 + S5 + S6, 0, 999.99)

Implementation: services/scoring_service.py. Tests: tests/test_scoring_service.py (35 tests).

### Product Comparison
- Maximum 3 products per comparison
- Color Match % = clamp((S2 + S4) / 300 × 100, 0, 100)
- Comparison table includes: name, brand, price, category, suitability, description, purchase links, color match %, face shape tips
- All labels in Thai language
- Face shape tips from FACE_SHAPE_TIPS_TH (Thai translations)

## 7. Product Color Data Pipeline

### Methodology
Colors are extracted from real product swatch images using K-Means clustering in CIELAB color space.

Reference: Griffen, A., Bailey, J., & Whaley, R. (2018). "Beauty Brawl", The Pudding. https://pudding.cool/2018/06/makeup-shades/

### Pipeline Steps (scripts/)
1. **download_swatches.py** — Downloads swatch images from official/retailer websites (Shopify CDN, Temptalia, beauty blogs)
2. **extract_product_colors.py** — Processes swatch images:
   - Load image, crop 15% margins (remove background)
   - K-Means clustering (5 clusters) in CIELAB space → dominant color
   - Classify tone via ITA (Chardon et al. 1991)
   - Classify undertone via hue angle (Xiao et al. 2017)
   - Generate shade_code: {tone}_{undertone} (e.g. "fair_cool", "medium_warm")
   - Export INSERT SQL with subquery product lookup
3. **seed_data.py** — Generates recommendation_rules from product_color_shades using shade_code

### Image Sources (115 swatch images)
- Romand: romandbeauty.com Shopify CDN (via .json API)
- Canmake: canmakeusa.com Shopify CDN, TokTokBeauty.com Shopify CDN
- 3CE: Lamcy Korea Shopify CDN (lamcy-kr.com)
- MAC: Temptalia.com, soundlysensiblebeauty blog
- Maybelline: Temptalia.com, makeupandbeautyforever blog, Nouveau Cheap blog
- Mistine: BigxShow Shopify CDN, Mistine official website
- Canmake Glow Fleur: alittledabintomakeup blog, Momoko Cosmetics Shopify CDN

### Color Conversion Path
sRGB [0-255] → normalize [0-1] → XYZ (D65 illuminant) → CIELAB (L*a*b*)
Using colour-science library: colour.sRGB_to_XYZ() → colour.XYZ_to_Lab()

## 14. Color Extraction Science

### K-Means + CIELAB Methodology
Product colors are extracted from real swatch images, NOT estimated or made up.

1. Load swatch image → crop 15% margins (remove white/black backgrounds)
2. Convert all pixels: sRGB → XYZ (D65) → CIELAB
3. Filter extreme values: L* < 15 or L* > 95 (background remnants)
4. K-Means clustering (5 clusters, 10 initializations) → find largest cluster
5. Cluster centroid = dominant product color in L*a*b*

### Tone Classification (ITA)
ITA = arctan((L* - 50) / b*) × (180/π)
| ITA Range | Classification |
|-----------|---------------|
| > 55° | fair (Very Light) |
| > 41° | light (Light) |
| > 28° | medium (Intermediate) |
| > 10° | tan (Tan/Mat) |
| > -30° | dark (Brown) |
| ≤ -30° | deep (Dark) |

Reference: Chardon, A., Cretois, I., & Hourseau, C. (1991). Skin colour typology and suntanning pathways. Int J Cosmet Sci, 13(4), 191-208.

### Undertone Classification (Hue Angle)
h_ab = atan2(b*, a*) in degrees [0, 360)
| Hue Angle | Classification |
|-----------|---------------|
| > 65° | warm (yellow-dominant) |
| < 50° | cool (red/pink-dominant) |
| 50–65° | neutral |

Reference: Xiao, K., et al. (2017). Characterising the variations in ethnic skin colours. Skin Res Technol, 23(1), 21-29.

### Color Distance (CIEDE2000)
For matching user's skin to palette centroids and product shades.
Reference: Sharma, G., Wu, W., & Dalal, E. N. (2005). The CIEDE2000 color-difference formula. Color Research & Application, 30(1), 21-30.
