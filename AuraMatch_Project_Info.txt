1. ภาพรวมโปรเจกต์
--------------------------------------------------------------------------------
AuraMatch คือแพลตฟอร์ม AI แนะนำเครื่องสำอาง โดยวิเคราะห์รูปหน้า สีผิว อันเดอร์โทน
และ Personal Color เพื่อแนะนำผลิตภัณฑ์ที่เหมาะสม

ฟีเจอร์หลัก:
- วิเคราะห์รูปทรงใบหน้า (Face Shape Detection)
- วิเคราะห์สีผิวและอันเดอร์โทน (Skin Tone / Undertone Analysis)
- จำแนก Personal Color Season (Spring/Summer/Autumn/Winter × 3 sub-types = 12 seasons)
- แนะนำเครื่องสำอางด้วยระบบ Multi-Signal Scoring (6 สัญญาณ)
- เปรียบเทียบสินค้า (สูงสุด 3 ชิ้น)
- สินค้าคล้ายกัน (TF-IDF + Cosine Similarity)
- แต่งหน้าบนรูป (Photo Editor) + ฟิลเตอร์ + Preset Looks
- แชท AI ด้วย Gemini
- ระบบ Affiliate tracking (Shopee, TikTok Shop, Lazada)
- User Behavior Analytics

7.1 ai_service.py — Face Analysis Pipeline
--------------------------------------------------------------------------------
หน้าที่: ประสานงานการวิเคราะห์ใบหน้าทั้งหมด

ฟังก์ชันหลัก:
- analyze_face(image_path, gender, db_session)
  * เรียก _run_facemesh() ครั้งเดียว → ได้ landmarks (468 จุด), bounding box, face crop
  * ส่ง face crop ไป detect_face_shape()
  * ส่ง bounding box ไป extract_skin_lab_from_bbox()
  * เรียก get_personal_color() เพื่อ match season

- detect_face_shape(rgb_image, facemesh_data)
  * CNN path (primary): ใช้ face crop + models_ai/face_shape_cnn.h5
  * Geometry fallback: ใช้ MediaPipe landmark ratios

- detect_skin_tone(rgb_image) → (tone, undertone)
  * แยกสีผิวหลักใน CIELAB
  * จำแนก tone จาก L* / undertone จาก a*, b*

- get_personal_color(skin_tone, undertone, skin_lab, db_session)
  * CIEDE2000 matching กับ ColorPalette references (primary)
  * Fallback: legacy lookup table

Geometry Decision Tree Thresholds:
  ratio = face_height / cheekbone_width  (landmarks 152↔10 / 234↔454)
  jaw_ratio = jaw_width / cheekbone_width  (landmarks 172↔397 / 234↔454)
  temple_ratio = temple_width / cheekbone  (landmarks 127↔356 / 234↔454)

  ratio > 1.5                          → oblong
  1.3 < ratio <= 1.5:
    jaw_ratio < 0.75                   → heart
    temple_ratio < 0.85               → diamond
    else                               → oval
  1.0 < ratio <= 1.3:
    jaw_ratio > 0.9                   → square
    jaw_ratio < 0.75                  → heart
    temple_ratio < 0.85              → diamond
    else                               → oval
  ratio <= 1.0:
    jaw_ratio > 0.9                   → square
    else                               → round
  no face detected                     → oval (default)

FACE_SHAPE_TIPS: ข้อมูล contouring & styling สำหรับแต่ละรูปหน้า


7.2 color_analysis_service.py — CIELAB Color Science
--------------------------------------------------------------------------------
หน้าที่: การวิเคราะห์สีทั้งหมดใน CIELAB color space

ฟังก์ชัน:
- rgb_to_lab(rgb_pixels) → CIELAB array
  * sRGB → XYZ (D65) → L*a*b* via colour-science library

- extract_skin_lab(rgb_image) → (L, a, b)
  * Crop center region
  * K-Means clustering (k=3)
  * Return centroid of largest cluster

- extract_skin_lab_from_bbox(rgb_image, bbox) → (L, a, b)
  * Crop cheek region (middle third of face bbox)
  * K-Means clustering (k=6)
  * Filter outlier pixels (L* < 20 or > 95)

- classify_tone_from_lab(L) → str
  * L > 78: fair
  * 68 < L <= 78: light
  * 58 < L <= 68: medium
  * 48 < L <= 58: tan
  * 38 < L <= 48: dark
  * L <= 38: deep

- classify_undertone_from_lab(a, b) → str
  * b > 20 AND a > 8: warm
  * b < 12 OR (a > 12 AND b < 16): cool
  * else: neutral

- load_palette_references(db_session) → list
  * Query ColorPalette → extract skin_reference centroids
  * Module-level cache

- match_season_by_delta_e(skin_lab, palette_refs) → (season, palette_id, confidence)
  * CIEDE2000 distance จาก skin ไปทุก palette reference
  * Return ที่ใกล้ที่สุด + confidence (50-99%)


7.3 scoring_service.py — Multi-Signal Recommendation Scoring
--------------------------------------------------------------------------------
หน้าที่: คำนวณคะแนนแนะนำสินค้า 6 สัญญาณ

Signal  Range      Description
------  ---------  -----------
S1      0-300      Rule Match Priority (จาก RecommendationRule.priority)
S2      0-300      CIEDE2000 Proximity (ความใกล้สีสินค้ากับ palette best_colors)
S3      0-150      Seasonal Match (ตรง personal color season = 150, ไม่มีข้อมูล = 75, ไม่ตรง = 0)
S4      -100 to 0  Avoid-Color Penalty (สินค้าใกล้สีควรหลีกเลี่ยง, trigger เมื่อ delta_E < 15)
S5      0-100      Feedback Aggregation (like/dislike ratio จากผู้ใช้ personal color เดียวกัน)
S6      0-50       Popularity (จำนวน favorites, cap ที่ 10)

Final = clamp(S1 + S2 + S3 + S4 + S5 + S6, 0, 999.99)

ฟังก์ชัน:
- hex_to_lab(hex_color) → (L, a, b)  [cached]
- compute_color_proximity(shades_lab, palette_lab) → 0-300
  * delta_E=0 → 300, delta_E>=50 → 0
- compute_avoid_penalty(shades_lab, avoid_lab) → -100 to 0
- compute_personal_color_score(product_pc, analysis_pc) → 0/75/150
- compute_final_score(s1, s2, s3, s4, s5, s6) → 0-999.99
- batch_feedback_scores(product_ids, personal_color, db) → dict {product_id: S5}
- batch_popularity_scores(product_ids, db) → dict {product_id: S6}


7.4 recommendation_service.py — TF-IDF Product Similarity

Text Tokenization: Regex แยก Thai (ก-๙) + English (a-z) words

Document Building (per product):

  - Description (weight 2x)
  - Category (weight 2x)
  - Brand (weight 2x)

ฟังก์ชัน:
- get_similar_products(target_product, all_products, top_n=6) → list[dict]
  * Return top N products with similarity_score (0.0-1.0)

11. Color Science Rules
================================================================================

กฎเหล่านี้บังคับสำหรับทุก code ที่เกี่ยวกับสีผิว:

1. ใช้ CIELAB (L*a*b*) color space เท่านั้น — ห้ามใช้ raw RGB
2. ใช้ CIEDE2000 (Delta E00) สำหรับ color distance — ห้ามใช้ CIE76 หรือ CIE94
3. จำแนก skin tone จาก L* (lightness) — ห้ามใช้ RGB brightness
4. จำแนก undertone จาก a* และ b* — ห้ามใช้ r-b difference
5. Season matching ต้องใช้ Delta E กับ palette skin_reference centroids
6. Color palette data ต้องเก็บใน CIELAB format {L, a, b}
7. ใช้ colour-science library เป็นหลัก (sRGB → XYZ D65 → L*a*b*)
8. Legacy fallback ใน get_personal_color() ยังเก็บ lookup dict ไว้


================================================================================
12. Product Comparison Rules
================================================================================

1. เปรียบเทียบได้สูงสุด 3 สินค้า (backend + frontend validate)
2. Color Match % = clamp((S2 + S4) / 300 * 100, 0, 100)
   - S2 = CIEDE2000 proximity (0-300)
   - S4 = avoid-color penalty (-100 to 0)
3. UI labels ทั้งหมดเป็นภาษาไทย
4. Face shape tips ใช้ FACE_SHAPE_TIPS_TH + FACE_SHAPE_NAMES_TH
5. ห้ามใช้ text-transform: uppercase ใน comparison table
6. Graceful degradation: Color Match + Face Shape Tips แสดงเฉพาะเมื่อมีผลวิเคราะห์

13. Authentication & Security
================================================================================

Auth Flow:
  1. POST /auth/login → { access_token (JWT), role }
  2. Frontend เก็บ token + role ใน localStorage
  3. Axios interceptor แนบ Authorization: Bearer {token} ทุก request
  4. Backend decode JWT → get user_id + role
  5. Protected routes ใช้ Depends(get_current_user)

JWT Config:
  - Algorithm: HS256
  - Expire: 1440 นาที (24 ชั่วโมง)
  - Secret: จาก environment variable SECRET_KEY

Password: bcrypt hash via passlib

AuthContext (React):
  - login(token, role) → set user + localStorage
  - logout() → clear user + localStorage
  - useAuth() hook → access auth state
